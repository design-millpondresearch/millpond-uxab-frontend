# Plan: Interactive WorkBench Pricing Page with Stripe Integration

## Overview
Replace the "Contact Us" CTAs on the WorkBench page with an interactive pricing table, add a dedicated pricing route, and integrate Stripe Embedded Elements for direct in-app checkout. The design takes inspiration from Apollo's clean, feature-comparison pricing layout while preserving the existing Mill Pond Research color palette and typography.

## User Requirements Summary
- **Billing cycle**: Monthly only
- **Page location**: Both inline on the WorkBench page AND a new dedicated `/pricing` route
- **Stripe approach**: Embedded Stripe Elements (requires backend API for PaymentIntents)
- **Most Popular badge**: Business tier
- **Enterprise CTA**: Opens a contact modal (not navigation to `/contact`)

## 3. Component Architecture

### PricingCard Props
```
tier: "personal" | "business" | "enterprise"
name: string
price: string
subtitle: string
features: string[]
ctaLabel: string
isPopular: boolean
onCtaClick: () => void
```

### Payment Flow (Personal / Business)
1. User clicks "Subscribe" on Personal or Business card
2. `PaymentModal` opens
3. Frontend calls `POST /api/create-payment-intent` with `{ tier: "personal" | "business" }`
4. Backend creates a Stripe PaymentIntent (or Subscription with first payment) and returns `clientSecret`
5. Frontend initializes `<Elements>` with the `clientSecret`
6. User enters card details via `<PaymentElement>`
7. Frontend calls `stripe.confirmPayment()`
8. On success: show confirmation and close modal; on error: display error message

### Enterprise Flow
1. User clicks "Contact Us" on Enterprise card
2. `EnterpriseModal` opens with a contact form
3. Form submission goes to HubSpot (reusing existing `useHubspotForm` hook) or a simple email handler

## 4. Stripe Backend Specification

### `POST /api/create-payment-intent`
**Request Body:**
```json
{
  "tier": "personal" | "business"
}
```

**Response:**
```json
{
  "clientSecret": "pi_xxx_secret_yyy"
}
```

**Backend Logic:**
- Map tier to amount: `personal = 24900` cents, `business = 79900` cents
- Create a Stripe PaymentIntent with the amount in USD, monthly recurring
- Return the `client_secret`

**Required Environment Variables:**
- `STRIPE_SECRET_KEY` (sk_live_... or sk_test_...)
- `STRIPE_PERSONAL_PRICE_ID` (optional, for subscription-based approach)
- `STRIPE_BUSINESS_PRICE_ID` (optional, for subscription-based approach)

### Phase 1b: Duplicate Header Fix on Pricing Page
**Problem**: `Pricing.jsx` has a hero section with "WorkBench Pricing" title and subtitle. The `<PricingTable />` component (with `inline=false`) then renders its own section header that also says "WorkBench Pricing" with nearly identical subtitle text, creating a jarring duplicate.

**Fix**:
- Add a `showHeader` prop to `PricingTable` (default `true`).
- When `showHeader={false}`, the component renders only the pricing cards grid and the token-usage footer note — no section title or subtitle.
- Update `Pricing.jsx` to pass `showHeader={false}` to `<PricingTable />` since the hero already provides the page heading.
- Keep `Workbench.jsx` unchanged — the inline `PricingTable` retains its header because there's no hero above it on that page.

**Files to modify**:
- `src/components/pricing/PricingTable.jsx` — add `showHeader` prop and conditional header rendering

## 8. Risks & Notes

- **Backend Requirement**: The project currently has no backend. We are using Vercel serverless functions (`api/` directory) so everything deploys in a single tenant. The `vercel.json` rewrite rule must be updated so `/api/*` does NOT fall through to the SPA catch-all.
- **Stripe Keys**: The user must provide `STRIPE_SECRET_KEY` (test and live) and confirm whether they are using Stripe Products/Prices or raw PaymentIntents.
- **Enterprise Modal**: Reuses the HubSpot form portal ID (`48049833`). If a different form is needed for Enterprise leads, the user should provide the form ID.
- **Payment Confirmation**: After successful payment, consider redirecting to a success page or showing an inline confirmation. Plan includes basic modal success state.

## 9. Deployment Considerations

- **Single Vercel Tenant**: Vercel supports serverless functions alongside a Vite SPA in the same project. The `api/` directory at the project root is automatically deployed as serverless endpoints. No second tenant or separate host is required.
- **Vercel Rewrites**: The existing `vercel.json` catch-all rewrite (`/(.*)` → `/index.html`) must be updated to exclude `/api/*` paths so Stripe endpoints are reachable.
- **Environment Variables**: Set `STRIPE_SECRET_KEY` and `STRIPE_PUBLISHABLE_KEY` in the Vercel dashboard under Project Settings → Environment Variables.
